# Security Policy

## Reporting a vulnerability

**Please do not open a public issue for security problems.**

Report privately through GitHub's [**private vulnerability reporting**](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability):
on the affected repository, go to the **Security** tab → **Report a vulnerability**.

Please include:

- A description of the issue and its impact.
- Steps to reproduce (a proof of concept if possible).
- Affected version(s) and environment.

You'll get an acknowledgement as soon as possible. Please give a reasonable window to fix the issue
before any public disclosure. Thank you for helping keep these projects safe.
