# Security Policy · Política de Segurança

<sub>🇬🇧 English · 🇧🇷 <a href="#-português">Português</a></sub>

## 🇬🇧 English

### Reporting a vulnerability

**Please do not open a public issue for security problems.**

Report privately through GitHub's [**private vulnerability reporting**](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability):
on the affected repository, go to the **Security** tab → **Report a vulnerability**.

Please include:

- A description of the issue and its impact.
- Steps to reproduce (a proof of concept if possible).
- Affected version(s) and environment.

You'll get an acknowledgement as soon as possible. Please give a reasonable window to fix the issue
before any public disclosure. Thank you for helping keep these projects safe.

## 🇧🇷 Português

### Reportando uma vulnerabilidade

**Por favor, não abra uma issue pública para problemas de segurança.**

Reporte de forma privada pelo [**relato privado de vulnerabilidade**](https://docs.github.com/pt/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
do GitHub: no repositório afetado, vá na aba **Security** → **Report a vulnerability**.

Inclua, por favor:

- Uma descrição do problema e do impacto.
- Passos para reproduzir (uma prova de conceito, se possível).
- Versão(ões) e ambiente afetados.

Você receberá uma confirmação o mais rápido possível. Por favor, dê um prazo razoável para a correção
antes de qualquer divulgação pública. Obrigado por ajudar a manter estes projetos seguros.
